# javalens-manager v0.10.2

Polish release. Same feature set as v0.10.1, with a cleaner icon and the form-state fixes you've been waiting for.

## Highlights

- **Icon iteration.** The cream-white outer ring (and the v0.10.1 dark-navy follow-up) is gone — the slate body now extends fully to the canvas edge, and the magnifying glass + J inside are enlarged ~14% to fill the freed space. Reads cleaner at small sizes against any dock or taskbar theme.
- **Discover button greys after success.** Re-clicking Discover at the same path is a no-op; the button now reflects that. Change the file path to re-enable.
- **Import-selected button now follows the actionable-only style.** Matches Save project / Save settings — blue gradient only when there's something to import, plain disabled style otherwise.
- **Workspace-import form fully resets on success.** Workspace file, candidate list, and selection all clear so the form is ready for the next operation.

## Upgrade notes

- **Drop-in replacement for v0.10.1.** No settings migration. Existing `settings.json`, `projects.json`, and registered services keep working unchanged.
- **No runtime change.** Still pulls `javalens-mcp` v1.2.1 from the configured release source by default (`hw1964/javalens-mcp`).
- **Sprint 10 work in progress.** The next minor release (v0.11.0) will introduce port-grouped service consolidation — multiple projects sharing one MCP service against one in-memory workspace. See `docs/sprint-10-backlog.md` for the design.
