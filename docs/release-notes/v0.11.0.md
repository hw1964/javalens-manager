This release is the manager half of [fork v1.5.0](https://github.com/hw1964/javalens-mcp/releases/tag/v1.5.0). The manager has been polling for fork updates since v0.10.3; existing installs pick up v1.5.0 automatically on next dashboard load.

## What's new in the manager

### Help docs reflect the new tool surface

Settings â†’ Help now documents the v1.5.0 tool consolidation:

- The 13 narrow `find_*` tools are gone from the registered surface.
- `find_pattern_usages(kind, query)` covers `annotation` / `instantiation` / `type_argument` / `cast` / `instanceof`.
- `find_quality_issue(kind, ...)` covers `naming` / `bugs` / `unused` / `large_classes` / `circular_deps` / `reflection` / `throws` / `catches`.
- `find_method_references` and the position-anchored search tools (`find_references`, `find_implementations`, `find_field_writes`, `find_tests`) stay separate.

The Help page also notes that fork **v1.5.1** will add five LTK structural refactoring tools (`move_class`, `move_package`, `pull_up`, `push_down`, `encapsulate_field`); these get picked up automatically when the fork tag lands.
