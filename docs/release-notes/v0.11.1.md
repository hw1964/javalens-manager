# javalens-manager v0.11.1 — Sprint 11 closeout (paired with fork v1.5.2)

Patch release paired with [fork v1.5.2](https://github.com/hw1964/javalens-mcp/releases/tag/v1.5.2). The substantive fix happens in the fork — JDT-UI preference defaults, code-template store, members-order cache, and change-validation data are now initialised by JavaLens itself, so `move_class` / `move_package` / `pull_up` / `push_down` work in headless mode without depending on a prior Eclipse session's `.metadata`. (`encapsulate_field` still requires a prior Eclipse run's templates due to an upstream JDT bug.) The manager side is a docs + screenshots refresh.

## Pair with javalens-mcp v1.5.2

The manager has been polling for fork updates since v0.10.3; existing installs pick up v1.5.2 automatically on next dashboard load. No re-deploy needed — the tool surface is unchanged from v1.5.1 (still 60 tools per workspace service).

## What's new in the manager

### Refreshed help.md and screenshots

- `Tool surface (fork v1.5.x)` section now reflects the cumulative 66 → 55 → 60 tool-count progression instead of just calling out v1.5.0.
- The "Known limitation in v1.5.1" paragraph is replaced with a v1.5.2 closeout note explaining the JDT-UI preference fix.
- All three screenshots (`dashboard.png`, `settings-top.png`, `settings-bottom.png`) re-captured against current builds — workspace-grouped dashboard, refreshed Settings layout including v0.11.0 polish.
- New Linux note on the "Use system tray" setting: vanilla GNOME (Fedora Workstation, Debian GNOME) needs `gnome-shell-extension-appindicator` installed for the tray icon to appear; Pop!_OS / Ubuntu 22.04+ / KDE / XFCE / Cinnamon / MATE work out of the box. Cross-links to the README's new "System tray on Linux" subsection.

## Compatibility

- Drop-in upgrade from `v0.11.0`. No `projects.json` schema change. No new Tauri commands. No Rust or Svelte logic changes.
- Existing MCP client configs continue to work as-is.
