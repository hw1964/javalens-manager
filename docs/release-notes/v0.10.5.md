# javalens-manager v0.10.5 — Workspace polish: bulk move, drag-drop, Diagnostics counts

A Sprint 10 polish drop turning the named-workspace model into a much faster day-to-day tool. No backend changes; no fork version change required (still targets fork `v1.3.0` and onward).

## What changed

### Multi-select + bulk action bar (G.1)

- Every project row gets a leading checkbox. Click to toggle, **shift-click** to extend a range from the previous toggle, **ctrl/cmd-click** behaves like a plain click (toggle, single row).
- When ≥ 1 project is selected, a top-of-list **bulk action bar** appears with:
  - **Move to workspace…** — pick an existing workspace or create a new one inline. Applies to every selected project in one action; dropping projects whose source workspace already matches is a no-op.
  - **Start selected** / **Stop selected** — fan out the per-project start/stop RPC over the selection.
  - **✕** — clear the selection.
- Selection is cross-workspace: select 3 projects from `workspace-A`, 2 from `workspace-B`, hit Move → all 5 land in the destination.
- Selected rows highlight with a subtle blue tint so the visual state is scannable across collapsed/expanded workspace cards.

### Drag-and-drop between workspaces (G.2)

- Project rows in the right pane are draggable. Grab any row and drop on:
  - A **workspace card header** in the right pane.
  - A **workspace row** in the left Workspaces list (handles the case where the destination card is collapsed or scrolled out of view).
- Drag visualizations: the source row dims to ~45 % opacity; the drop target shows a 2 px accent outline + tinted background.
- If you drag a row that's part of an active multi-selection, the **whole selection** moves. Dragging an unselected row carries just that one row and leaves the selection intact.
- Native cancel works: pressing Esc or dropping outside a target reverts cleanly.

### Settings → Diagnostics workspace counts (G.3)

- The Diagnostics card on the Settings page now shows two extra cells alongside the file-path entries:
  - **Workspaces: N (M running)** — total workspaces and how many have at least one runtime up.
  - **Project count** — total projects across all workspaces.
- Pure client-side derivation, updates as you add/remove projects without any reload.

## Compatibility

- No `projects.json` schema change. Drop-in upgrade from `v0.10.4`; downgrade is also safe (the new fields are pure derived state).
- No fork version bump. Still works against javalens-mcp `v1.3.0` (and `v1.4.0` once Sprint 11 ships).
- No new Tauri commands; bulk move reuses the existing `set_project_workspace` RPC.
