# javalens-manager v0.12.0 — Tray menu lifecycle controls (paired with fork v1.6.0)

Sprint 12 cutover. The tray menu grows from "Show / Stop all / Quit" into a real workspace control panel: per-workspace toggle entries with live status icons, plus a "Start all services" peer to "Stop all services". Paired with [fork v1.6.0](https://github.com/hw1964/javalens-mcp/releases/tag/v1.6.0), which adds two Ring 1 MCP tools (`compile_workspace`, `run_tests`).

## What's new in the manager

### Tray menu rework

Old menu:
```
Show
Stop all services
Quit
```

New menu:
```
Show
─────────────────────────
[●] jats                       (per-workspace toggle; click starts/stops)
[●] internal-tools
[○] legacy-suite
─────────────────────────
Start all services
Stop all services
─────────────────────────
Quit
```

Status icon colors map to runtime phase:

- 🟢 **Running** — `#22C55E`
- 🟡 **Starting** — `#F59E0B`
- 🔴 **Failed** — `#EF4444`
- ⚪ **Stopped** — `#6B7280`

Icons are 16×16 RGBA, **generated procedurally at startup** (no static asset files committed) — same approach as the existing tray-icon glyph drawing in `lib.rs`. The OS scales for HiDPI from the single 16×16 source.

**Click semantics:** clicking a workspace entry **toggles** that workspace — start when current phase is Stopped/Failed, stop when Running/Starting. The state-keyed action means the user's mental model is "click to toggle" without needing visible "Start" / "Stop" prefixes.

**Live refresh:** the menu rebuilds every ~5 s in a background thread and immediately after every tray-driven start/stop. So external state changes (e.g. a workspace's javalens process getting killed from the shell) propagate to the per-workspace status icons automatically.

### Fork v1.6.0 verification tools (manager picks them up automatically)

Two new MCP tools, available to agents the moment fork v1.6.0 lands via the manager's existing release-poller:

- **`compile_workspace`** — incremental Java build over every loaded project + aggregated `IMarker` problem markers in one call. Replaces the per-file `get_diagnostics` walk for "did my refactor compile?" questions.
- **`run_tests`** — JUnit 4 / 5 / TestNG via JDT-LTK's launching delegate. Scope is `method` / `class` / `package`. Returns parsed pass/fail/skipped counts with per-failure stack traces and stdout/stderr tail.

Per-workspace tool count grows from **60 → 62**.

## Compatibility

- Drop-in upgrade from `v0.11.1`. No `projects.json` schema change. No new Tauri commands.
- One new `ManagerService` API: `workspace_status_summary()` plus `toggle_workspace(name)`. Exposed for the tray; the dashboard continues to use the existing per-project APIs.
- 4 new Rust unit tests (`aggregate_workspace_phase_*`).

## System tray on Linux

Same caveat as v0.11.0 — the tray relies on a StatusNotifierItem / AppIndicator host:

- **Pop!_OS, Ubuntu 22.04+, KDE Plasma, XFCE, Cinnamon, MATE** — works out of the box, nothing to install.
- **Vanilla GNOME (Fedora Workstation, Debian GNOME)** — install `gnome-shell-extension-appindicator` once. On Fedora: `sudo dnf install gnome-shell-extension-appindicator && gnome-extensions enable appindicatorsupport@rgcjonas.gmail.com`. On Debian: `sudo apt install gnome-shell-extension-appindicator`. Log out and back in after enabling.

The new per-workspace entries have the same dependency — if the existing v0.11.x tray was visible, the v0.12.0 tray menu and its status icons will be too.
