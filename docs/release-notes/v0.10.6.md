# javalens-manager v0.10.6 — Help refresh + workspace UX polish

A polish drop on top of v0.10.5. The user-facing capabilities (workspaces, multi-select bulk move, drag-drop, Diagnostics workspace counts) shipped in v0.10.5 — this release fixes the rough edges and rewrites the in-app Help to lead with the new workspace-centric model.

> **What is javalens-manager now?** A control plane for **named workspaces of Java projects**, each running as one shared JavaLens MCP service. The point: AI agents get the same IDE-grade understanding of a Java codebase that a human developer gets in Eclipse or IntelliJ — call hierarchies, type hierarchies, references, refactorings, build classpath, JDK semantics. **Java agents on steroids.**

## What changed

### Help docs rewritten around workspaces

[`src/assets/help.md`](../../src/assets/help.md):
- Opening pitch leads with workspaces and the agent-IDE-features story.
- New **Workspaces — the core concept** section that consolidates what used to be a footnote, including status-lamp colors, tool budget, and migration from v0.10.3.
- Dashboard section documents the current UX: Workspaces card on the left with right-click + hover icons; grouped Managed Projects on the right with right-click context menus, multi-select bulk action bar, and drag-and-drop between workspaces.
- Settings section drops the obsolete *Permitted project ports* paragraph (workspaces replaced ports in v0.10.4) and adds the **Workspaces** / **Project count** entries that now live in the Diagnostics card.
- Quick-reference table updated to point at the new flows.

### Workspace expand/collapse is instant

The right-pane workspace cards previously mounted/unmounted ~30 nested project rows on every chevron click — visibly slow when expanding a busy workspace. The rows are now always rendered; toggling collapses via CSS `display: none` instead of `{#if !collapsed}`. First click → instant fold or unfold.

### First-click on the workspace toggle now works

For workspaces that auto-collapse (default for the *stopped* phase), the first chevron click previously appeared to do nothing — only the second click took effect. The toggle was flipping the *override* slot whose `!undefined` value happened to match the auto-default; the resolved visible state didn't change. The toggle now flips the resolved visible state directly, so one click is enough.

### Workspace status lamps differentiate by phase

Every status lamp variant was rendering the same slate-gray (`#cbd5e1`). They now actually carry phase color:

- **stopped** → slate `#94a3b8`
- **starting** / mixed → amber `#fbbf24`
- **running** → emerald `#34d399`
- **failed** → coral `#f87171`

Both the lamp on each row in the left **Workspaces** card and the badges in the right pane pick up the phase automatically.

### Project-row layout cleaned up

When the multi-select checkbox column landed in v0.10.5, the project row's `flex` `space-between` layout split into checkbox-far-left, name-floating-in-the-middle, actions-far-right — visually broken on wide windows. Replaced with an explicit CSS grid (`auto 1fr auto`):

- Column 1 (`auto`): 1 rem checkbox with explicit `accent-color: #60a5fa` so the checked state actually reads against the dark theme.
- Column 2 (`1fr`): project name on top, path beneath, both left-aligned.
- Column 3 (`auto`): refresh icon, status badge, Start / Stop / Delete.

Narrow-viewport override drops to a 2-column layout with the action column wrapping below.

## Compatibility

- No `projects.json` schema change. Drop-in upgrade from `v0.10.5`; downgrade is also safe.
- No fork version bump. Targets javalens-mcp `v1.4.0` (current) or `v1.3.0`.
- No new Tauri commands.
